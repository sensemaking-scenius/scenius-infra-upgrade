# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Jekyll-based dashboard and archive for the Sensemaking Scenius community. Deployed as a GitHub Pages project site at `/scenius-infra-upgrade/`. Tracks async deliberation sessions, live workshop summaries, and community artifacts (proposals, specs).

**Live:** https://sensemaking-scenius.github.io/scenius-infra-upgrade/

## Architecture

Static site with three Jekyll collections and no build tooling beyond Jekyll itself:

- `_sessions/` — auto-synced from Harmonica via GitHub Actions (every 6 hours) using `harmonica-sync` CLI. Config in `harmonica.config.json`.
- `_workshops/` — manually written markdown summaries from live calls.
- `_artifacts/` — proposals, specs, and community documents (manually added).

Four layouts in `_layouts/`:
- `default.html` — base layout containing ALL CSS (inline `<style>` block, ~750 lines) and HTML shell. There are no external stylesheets.
- `session.html` — parses session markdown into a card-grid dashboard via client-side JS, with collapsible full transcript.
- `workshop.html` — detail page with keyword chips, participant grid.
- `artifact.html` — detail page with artifact_type badge, source_url link, author chips.

`index.html` — main page with stats bar, consensus block, contributor chips, and a combined activity feed merging all three collections sorted by date (using Liquid string encoding: `date::type::index` tuples).

## Design System

All styles live in `_layouts/default.html`. The aesthetic uses:
- **Fonts:** Calistoga (serif, headings) + DM Sans (sans, body) via Google Fonts
- **Colors:** CSS variables — `--mint` (#81D7C0), `--pink` (#CF5CD7), `--cream`, `--ink` variants
- **Shapes:** `border-radius: 10-12px` for cards, `999px` for pills/chips
- **Effects:** SVG noise texture overlay (`body::before`), gradient blobs (`body::after`), glass-morphism stats bar (`backdrop-filter: blur`)
- **Animation:** `fadeUp`, `fadeIn`, `scaleIn` keyframes with staggered `nth-child` delays on activity list items

When making visual changes, preserve this aesthetic. Use existing CSS variables rather than introducing new colors.

## Session Sync

`harmonica.config.json` configures which sessions to sync. The GitHub Action (`.github/workflows/sync-sessions.yml`) runs `npx harmonica-sync` and commits changes to `_sessions/`. Requires `HARMONICA_API_KEY` repo secret.

Manual sync: `HARMONICA_API_KEY=hm_live_... npx harmonica-sync`

## Frontmatter Schemas

**Sessions** (auto-generated by harmonica-sync):
```yaml
title, date, session_id, participants (count), status (active|completed), tags[]
```

**Workshops** (manual):
```yaml
title, date, type: workshop, duration (minutes), participants_list[], keywords[]
```

**Artifacts** (manual):
```yaml
title, date, type: artifact, artifact_type (proposal|whiteboard|spec|...), source_url, authors[], keywords[]
```

## Project Docs

`docs/plans/` contains facilitation audits and session response analyses. These are internal research docs, not published to the site.

## Content Guidelines

- The **consensus block** on the main page (`index.html`) is currently hardcoded and requires manual updates. Planned: auto-generated consensus via `harmonica-sync` synthesis step (HAR-338) — will read all collections and write to `_data/consensus.yml`.
- Contributors are computed automatically from `participants_list` (workshops) and `authors` (artifacts) — not from sessions.
- Session content is auto-formatted by `session.html` JS: H2 headings become card titles, certain sections are displayed as dashboard cards (Core Vision, MVP Scope, etc.), others are skipped.

## Deployment

Push to `main` triggers GitHub Pages build. No custom build step — Jekyll runs natively on GitHub Pages. All URLs must use `{{ ... | relative_url }}` filter for the `/scenius-infra-upgrade/` base path.
